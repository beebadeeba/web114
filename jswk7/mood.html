Questions & Answers

Why is the JavaScript textContent property safer than innerHTML?

The textContent property is safer because it treats any inserted text as plain text instead of HTML. This prevents malicious scripts from being executed, reducing the risk of cross‑site scripting (XSS) attacks.

If you use an HTML text box with an attribute like id="text"and a corresponding DOM property using document.querySelector("#text"), how do the two relate?

The id attribute uniquely identifies an element in the HTML. The querySelector method retrieves that element from the DOM, giving JavaScript a reference to interact with it.

What would happen to the text box with id="text" if you added text.textContent = "Hello!"?

The text inside the element would change to display "Hello!". If it is an input element, it would not affect the value—only elements that display text (like paragraphs or divs) update visually.
